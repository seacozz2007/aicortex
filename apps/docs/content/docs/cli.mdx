The AICortex CLI mirrors almost everything the Web UI can do (create [issues](/issues), assign [agents](/agents), start the [daemon](/daemon-runtimes), and more). This page lists every top-level command with a one-line description. For the full set of flags and examples, run `aicortex <command> --help`.

## Getting authenticated

Run this the first time you use the CLI to obtain a **personal access token (PAT)**:

```bash
aicortex login
```

Your browser opens automatically. After you approve in the web app, the CLI saves the PAT (prefixed with `mul_`) to `~/.aicortex/config.json`. Every subsequent command authenticates with that PAT.

<Callout type="tip">
For CI or headless environments, skip the browser flow: create a PAT in the web app under **Settings → Personal Access Tokens**, then run `aicortex login --token <mul_...>` to supply it directly.
</Callout>

For the difference between token types, see [Authentication and tokens](/auth-tokens).

## Auth and setup

| Command | Purpose |
|---|---|
| `aicortex login` | Log in and save a PAT |
| `aicortex auth status` | Show current login status, user, and workspace |
| `aicortex auth logout` | Clear the local PAT |
| `aicortex setup cloud` | One-shot setup for AICortex Cloud (login + install daemon) |
| `aicortex setup self-host` | One-shot setup for a self-hosted backend |

## Workspaces and members

| Command | Purpose |
|---|---|
| `aicortex workspace list` | List every workspace you can access |
| `aicortex workspace get <slug>` | Show details for one workspace |
| `aicortex workspace members` | List members of the current workspace |
| `aicortex workspace update <id> --name "..." [--description "..."] [--context "..."] [--issue-prefix "..."]` | Update workspace metadata (admin/owner). Long fields accept `--description-stdin` / `--context-stdin`. |

## Issues and projects

<Callout type="info">
`list` commands (`aicortex issue list`, `autopilot list`, `project list`, etc.) print short, copy-paste-ready IDs by default — issue keys like `MUL-123` for issues, short UUID prefixes for the rest. The `<id>` argument on the follow-up commands below accepts either the short ID or the full UUID, so the typical flow is `aicortex issue list` → copy the key → `aicortex issue get MUL-123`. Pass `--full-id` to a list command when you need the canonical UUID.
</Callout>

| Command | Purpose |
|---|---|
| `aicortex issue list` | List issues (prints copy-paste-ready issue keys) |
| `aicortex issue get <id>` | Show a single issue (accepts an issue key or a UUID) |
| `aicortex issue create --title "..."` | Create a new issue |
| `aicortex issue update <id> ...` | Update an issue (status, priority, assignee, etc.) |
| `aicortex issue assign <id> --agent <slug>` | Assign to an agent (triggers a task immediately) |
| `aicortex issue status <id> --set <status>` | Shortcut to change status |
| `aicortex issue search <query>` | Keyword search |
| `aicortex issue runs <id>` | Show agent runs on an issue |
| `aicortex issue rerun <id>` | Re-enqueue a fresh task for the issue's current agent assignee |
| `aicortex issue comment <id> ...` | Nested: view / post comments |
| `aicortex issue subscriber <id> ...` | Nested: subscribe / unsubscribe |
| `aicortex project list/get/create/update/delete/status` | Project CRUD |

## Agents and skills

| Command | Purpose |
|---|---|
| `aicortex agent list` | List the workspace's agents |
| `aicortex agent get <slug>` | Show an agent's configuration |
| `aicortex agent create ...` | Create an agent |
| `aicortex agent update <slug> ...` | Update an agent |
| `aicortex agent archive <slug>` | Archive |
| `aicortex agent restore <slug>` | Restore an archived agent |
| `aicortex agent tasks <slug>` | Show an agent's task history |
| `aicortex agent skills ...` | Nested: attach / detach skills |
| `aicortex skill list/get/create/update/delete` | Skill CRUD |
| `aicortex skill import ...` | Import a skill from GitHub, ClawHub, or the local machine |
| `aicortex skill files ...` | Nested: manage a skill's files |

## Squads

| Command | Purpose |
|---|---|
| `aicortex squad list` | List squads in the workspace |
| `aicortex squad get <id>` | Show a single squad |
| `aicortex squad create --name "..." --leader <agent>` | Create a squad (owner / admin) |
| `aicortex squad update <id> ...` | Update name, description, instructions, leader, or avatar |
| `aicortex squad delete <id>` | Archive (soft-delete) — transfers assigned issues to the leader |
| `aicortex squad member list/add/remove <squad-id>` | Manage squad members |
| `aicortex squad activity <issue-id> <action\|no_action\|failed> --reason "..."` | Used by squad leader agents to record an evaluation per turn |

See [Squads](/squads) for the full model.

| Command | Purpose |
|---|---|

## Daemon and runtimes

| Command | Purpose |
|---|---|
| `aicortex daemon start` | Start the daemon (background by default; add `--foreground` to run in the foreground) |
| `aicortex daemon stop` | Stop the daemon |
| `aicortex daemon restart` | Restart the daemon |
| `aicortex daemon status` | Check whether the daemon is online and its concurrency |
| `aicortex daemon logs` | View daemon logs |
| `aicortex runtime list` | List runtimes in the current workspace |
| `aicortex runtime usage` | Show resource usage |
| `aicortex runtime activity` | Recent activity log |
| `aicortex runtime update <id> ...` | Update a runtime's configuration |

## Miscellaneous

| Command | Purpose |
|---|---|
| `aicortex repo checkout <url>` | Clone a repo locally for agents to use |
| `aicortex config` | View or edit local CLI configuration |
| `aicortex version` | Print the CLI version |
| `aicortex update` | Upgrade the CLI to the latest release |
| `aicortex attachment download <id>` | Download an attachment from an issue or comment |

## Getting full flags

```bash
aicortex issue create --help
aicortex agent update --help
```

v2 will ship a dedicated detailed reference page for each command.
