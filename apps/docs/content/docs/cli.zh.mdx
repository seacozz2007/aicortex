AICortex CLI 把 Web UI 能做的事几乎全部搬到了命令行上（创建 [issue](/issues)、分配 [智能体](/agents)、启动 [守护进程](/daemon-runtimes) 等等）。这一页把所有顶级命令列出来，每条配一句用途。完整 flag 和示例用 `aicortex <命令> --help` 查。

## 认证入口

第一次用 CLI 时先登录，拿一个**个人访问令牌（Personal Access Token，PAT）**：

```bash
aicortex login
```

浏览器会自动打开，你在 Web 端同意后，CLI 把 PAT（`mul_` 前缀）保存到 `~/.aicortex/config.json`。此后所有命令都会自动用这个 PAT 认证。

<Callout type="tip">
CI / 无浏览器环境跳过浏览器流程：先在 Web 端 **Settings → Personal Access Tokens** 创建一个 PAT，然后 `aicortex login --token <mul_...>` 直接填入。
</Callout>

Token 类型的详细区分见 [认证与令牌](/auth-tokens)。

## 认证与初始化

| 命令 | 用途 |
|---|---|
| `aicortex login` | 登录并保存 PAT |
| `aicortex auth status` | 查看当前登录状态、用户、工作区 |
| `aicortex auth logout` | 清除本地 PAT |
| `aicortex setup cloud` | AICortex Cloud 一键初始化（登录 + 装 daemon） |
| `aicortex setup self-host` | 自部署后端的一键初始化 |

## 工作区和成员

| 命令 | 用途 |
|---|---|
| `aicortex workspace list` | 列出你有权访问的所有工作区 |
| `aicortex workspace get <slug>` | 查看一个工作区的详情 |
| `aicortex workspace members` | 列出当前工作区的成员 |
| `aicortex workspace update <id> --name "..." [--description "..."] [--context "..."] [--issue-prefix "..."]` | 修改 workspace 元数据（admin/owner 权限）。长文本可用 `--description-stdin` / `--context-stdin`。 |

## Issue 和 Project

<Callout type="info">
`list` 类命令（`aicortex issue list`、`autopilot list`、`project list` 等）表格里默认显示**可直接复制**的短 ID：issue 是 key（如 `MUL-123`），其余资源是 UUID 短前缀。下面表格里的 `<id>` 同时接受短 ID 和完整 UUID，所以典型用法是 `aicortex issue list` → 复制 key → `aicortex issue get MUL-123`。需要完整 UUID 时给 `list` 加 `--full-id`。
</Callout>

| 命令 | 用途 |
|---|---|
| `aicortex issue list` | 列出 issue（默认显示可复制的 issue key） |
| `aicortex issue get <id>` | 查看单条 issue（接受 issue key 或 UUID） |
| `aicortex issue create --title "..."` | 创建新 issue |
| `aicortex issue update <id> ...` | 修改 issue（状态、优先级、分配人等） |
| `aicortex issue assign <id> --agent <slug>` | 分配给智能体（立即触发任务） |
| `aicortex issue status <id> --set <status>` | 快捷改状态 |
| `aicortex issue search <query>` | 关键字搜索 |
| `aicortex issue runs <id>` | 查看 issue 上智能体跑过的任务 |
| `aicortex issue rerun <id>` | 给该 issue 当前的智能体分配人重新创建一条任务 |
| `aicortex issue comment <id> ...` | 嵌套：看 / 发评论 |
| `aicortex issue subscriber <id> ...` | 嵌套：订阅 / 取消订阅 |
| `aicortex project list/get/create/update/delete/status` | Project CRUD |

## 智能体和 Skill

| 命令 | 用途 |
|---|---|
| `aicortex agent list` | 列出工作区的智能体 |
| `aicortex agent get <slug>` | 查看智能体配置 |
| `aicortex agent create ...` | 创建智能体 |
| `aicortex agent update <slug> ...` | 修改智能体 |
| `aicortex agent archive <slug>` | 归档 |
| `aicortex agent restore <slug>` | 恢复归档的智能体 |
| `aicortex agent tasks <slug>` | 查看智能体的任务历史 |
| `aicortex agent skills ...` | 嵌套：挂载 / 卸载 Skill |
| `aicortex skill list/get/create/update/delete` | Skill CRUD |
| `aicortex skill import ...` | 从 GitHub / ClawHub / 本机导入 Skill |
| `aicortex skill files ...` | 嵌套：管理 Skill 的文件 |

## 小队

| 命令 | 用途 |
|---|---|
| `aicortex squad list` | 列出工作区里的小队 |
| `aicortex squad get <id>` | 查看一个小队 |
| `aicortex squad create --name "..." --leader <agent>` | 创建小队（owner / admin）|
| `aicortex squad update <id> ...` | 修改名字、描述、instructions、队长、头像 |
| `aicortex squad delete <id>` | 归档（软删除）—— 同时把分配给小队的 issue 转给队长 |
| `aicortex squad member list/add/remove <squad-id>` | 管理小队成员 |
| `aicortex squad activity <issue-id> <action\|no_action\|failed> --reason "..."` | 队长智能体每轮结束时调用，记录 evaluation |

完整模型见 [小队](/squads)。

## 守护进程和运行时

| 命令 | 用途 |
|---|---|
| `aicortex daemon start` | 启动 daemon（默认后台；加 `--foreground` 前台跑）|
| `aicortex daemon stop` | 停止 daemon |
| `aicortex daemon restart` | 重启 daemon |
| `aicortex daemon status` | 查看 daemon 是否在线 + 并发情况 |
| `aicortex daemon logs` | 查看 daemon 日志 |
| `aicortex runtime list` | 列出当前工作区的 runtime |
| `aicortex runtime usage` | 查看资源使用情况 |
| `aicortex runtime activity` | 近期活动记录 |
| `aicortex runtime update <id> ...` | 更新 runtime 配置 |

## 杂项

| 命令 | 用途 |
|---|---|
| `aicortex repo checkout <url>` | 把 repo 拉到本地以供智能体使用 |
| `aicortex config` | 查看 / 修改 CLI 本地配置 |
| `aicortex version` | 显示 CLI 版本 |
| `aicortex update` | 升级 CLI 到最新版 |
| `aicortex attachment download <id>` | 下载 issue / 评论的附件 |

## 查完整 flag

每条命令都支持 `--help`：

```bash
aicortex issue create --help
aicortex agent update --help
```

v2 会给每条命令一个独立的详细 reference 页。
